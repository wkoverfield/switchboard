[ 02 ] · the loop

Three commands to a governed agent

Start in a repo where your agents already work. Switchboard meets your existing MCP config where it is.

01 / see it

scan, then import

$ switchboard scan
high claude:github
  token in plaintext .mcp.json
$ switchboard import --write
  token → OS keychain
  backup + exact rollback kept

Every server and token your agents can already touch, including the routes that bypass everything.

02 / scope it

grant a pass

$ switchboard grant --for 4h
SWITCHBOARD · PASS GRANTED
  → github_ci_*
  everything else denied
  expires in 4h

Named tools reachable, everything else denied, gone on its own. Revoke early any time.

03 / watch it hold

denied, on the record

agent → delete_file
SWITCHBOARD · DENIED
  never reached the provider
$ switchboard audit verify
Chain: OK

Denials, approvals, and every routed call land in a hash-chained local audit log you can verify.

[ 03 ] · what this pass does not cover

Trust tooling that overstates itself is worse than none.

These limits are in the README, the threat model, and the product's own output.

It is not a sandbox.

Switchboard governs the paths routed through it: its MCP endpoints and switchboard run. Raw shell, provider CLIs, browser sessions, and direct MCP routes bypass it. switchboard scan reports those routes so you can close them or accept them deliberately.

No pass, no policy.

With no active pass, configured profiles are served without tool policy. Enforcement is opt-in by design today; a default-deny mode is on the roadmap, and we will not claim it before it ships.

Revocation is next-call.

Revoking a pass stops the next routed call. A call already executing completes. If an in-flight action must die, revoke the token at the provider.

It is alpha software.

Single developer, local-first, conservative claims. The pass and audit contracts are versioned JSON; the threat model states every accepted risk.

[ 04 ] · diligence

Built to be diligenced

The security story is written down, adversarially, with code references. Read it before you trust us. Especially then.

threat-model.md A real threat model, not a posture note STRIDE analysis of every surface: what enforcement binds, what it cannot, revocation semantics, socket assumptions, and a ten-item accepted-risks list. Read it.
audit verify Tamper-evident audit log Every routed call, denial, and approval decision is hash-chained JSONL on your machine. Verification reports the exact line where a chain breaks. What it proves and what it cannot.
secretRef Secrets stay in the OS keychain Config carries named refs, never values. Weaker backends require an explicit unsafe opt-in. Values are injected only into the upstream that declared them, at spawn, and redacted everywhere else.
child ⊆ parent Delegation that cannot widen A lead agent can hand narrower passes to subagents; children are validated as strict subsets of the parent, and parent deny lists are inherited.
[ 05 ] · for agents

Installable by the agent reading this

The docs ship as an MCP server, and llms.txt is kept current. An agent can evaluate Switchboard, read the threat model, and set it up without leaving its own tools.

$ claude mcp add switchboard-docs -- npx -y @switchboard-mcp/docs-mcp
# tools: list_docs · read_doc · search_docs

$ npx -y @switchboard-mcp/cli@latest scan
