# Switchboard Product Docs

Start with `roadmap.md`, then `mandate-strategy.md`.

The files in `source/` are preserved planning documents from the original
Switchboard planning thread. Treat them as source material. Treat `roadmap.md`
as the current working plan.

Current product framing:

- Layer 1: easy repo-aware MCP/environment setup.
- Layer 2: optional task-scoped mandates.
- Layer 3: future mandate delegation chains.
- Layer 4: harness integration through CLI and JSON surfaces.

These are conceptual product layers, not build order. Build harness-friendly
active-mandate surfaces before child mandate/delegation enforcement.

`mandate-strategy.md` supersedes the older generic MCP profile-router thesis
where they conflict, but mandates should not be mandatory for simple setup.
